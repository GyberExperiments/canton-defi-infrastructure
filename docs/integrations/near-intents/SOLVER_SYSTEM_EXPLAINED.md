## 🎯 Что такое Solver System?

**Solver System** - это автоматизированная система, которая **исполняет (fulfill) пользовательские интенты** в протоколе NEAR Intents.

### Простыми словами:

**БЕЗ Solver System:**
1. Пользователь создает intent: "Хочу обменять 100 NEAR на USDT"
2. Intent висит в блокчейне и ждет...
3. Никто не исполняет его 😢

**С Solver System:**
1. Пользователь создает intent: "Хочу обменять 100 NEAR на USDT"
2. Solver видит новый intent
3. Solver проверяет: могу ли я исполнить это с прибылью?
4. Если да → исполняет через DEX
5. Пользователь получает USDT, solver получает комиссию ✅

## 💰 Как Solver Зарабатывает?

### Пример:

**Intent пользователя:**
- Отдает: 100 NEAR
- Хочет получить: минимум 10,000 USDT
- Deadline: через 1 час

**Действия Solver:**

1. **Проверяет цену на REF Finance:**
   - Текущая цена: 1 NEAR = 101 USDT
   - Значит за 100 NEAR можно получить: 10,100 USDT

2. **Рассчитывает прибыльность:**
   ```
   Получу за исполнение: 10,100 USDT
   Должен вернуть пользователю: 10,000 USDT (min_receive)
   Прибыль: 10,100 - 10,000 = 100 USDT
   Gas cost: ~0.01 NEAR = ~1 USDT
   Чистая прибыль: 100 - 1 = 99 USDT ✅
   ```

3. **Исполняет intent:**
   - Обменивает 100 NEAR → 10,100 USDT через REF Finance
   - Отправляет 10,000 USDT пользователю (минимум)
   - Остаток 100 USDT - это его комиссия

4. **Результат:**
   - ✅ Пользователь получил желаемый минимум (10,000 USDT)
   - ✅ Solver заработал 100 USDT
   - ✅ Intent исполнен и закрыт

## 🔄 Жизненный Цикл Intent с Solver

```
1. Пользователь создает Intent
   └─> Smart Contract сохраняет intent в "pending" статусе

2. Solver Node сканирует новые intents (каждые 2-5 секунд)
   └─> Находит новый intent

3. Solver проверяет прибыльность
   ├─> НЕ прибыльно → Пропускает
   └─> Прибыльно → Переходит к исполнению

4. Solver исполняет intent
   ├─> Обменивает через DEX
   ├─> Отправляет пользователю минимум
   └─> Получает прибыль

5. Smart Contract обновляет статус intent
   └─> Статус: "filled" ✅
```

## 🛠️ Технические Компоненты

### 1. Intent Monitor (Мониторинг Интентов)

**Задача:** Постоянно сканировать блокчейн на наличие новых pending intents

**Реализация:**
```typescript
class IntentMonitor {
  // Polling каждые 2 секунды
  async checkNewIntents() {
    const pendingIntents = await contract.get_pending_intents()
    for (const intent of pendingIntents) {
      await this.processIntent(intent)
    }
  }
}
```

### 2. Profitability Calculator (Калькулятор Прибыльности)

**Задача:** Рассчитать, будет ли исполнение intent прибыльным

**Формула:**
```
Profit = (ActualOutput - MinReceive) - GasCosts

Если Profit > 0 → Выгодно исполнять
Если Profit <= 0 → Не выгодно, пропускаем
```

**Факторы:**
- Текущая цена на DEX
- Минимальная сумма которую нужно вернуть пользователю
- Gas costs для транзакций
- Комиссии DEX

### 3. Intent Executor (Исполнитель Интентов)

**Задача:** Физически исполнить intent через DEX

**Шаги:**
1. Получить ликвидность с DEX
2. Обменять токены
3. Отправить пользователю min_receive
4. Подтвердить исполнение в контракте

## 🎯 Преимущества Solver System

### Для Пользователей:
- ✅ **Автоматическое исполнение** - не нужно ждать
- ✅ **Лучшая цена** - solvers конкурируют
- ✅ **Защита от slippage** - min_receive гарантирован
- ✅ **Trustless** - через смарт-контракт

### Для DEX (Нас):
- ✅ **Больше объемов** - solvers исполняют больше интентов
- ✅ **Лучшая ликвидность** - больше активности
- ✅ **Комиссии** - можем брать комиссию с solvers
- ✅ **Конкурентоспособность** - быстрые исполнения

### Для Solvers:
- ✅ **Прибыль** - зарабатывают на разнице цен
- ✅ **Автоматизация** - не нужно вручную искать интенты
- ✅ **Масштабируемость** - могут исполнять много интентов параллельно

## ⚠️ Важные Моменты

### Безопасность:
- ✅ Solver должен иметь достаточный баланс для исполнения
- ✅ Нужно проверять deadline (не исполнять истекшие)
- ✅ Валидация параметров перед исполнением
- ✅ Обработка ошибок (если DEX недоступен)

### Экономика:
- ⚠️ Gas costs могут "съесть" прибыль на маленьких суммах
- ⚠️ Нужен минимальный threshold прибыльности
- ⚠️ Рыночная волатильность может изменить прибыльность

### Производительность:
- ⚠️ Polling слишком часто → большой расход на RPC
- ⚠️ Polling слишком редко → пропустим интенты
- ✅ Оптимально: 2-5 секунд

## 🎯 Итого

**Solver System** - это "мозг" автоматического исполнения интентов. Без него интенты будут висеть в блокчейне без исполнения. С ним - пользователи получают быстрые и выгодные обмены, а solvers зарабатывают на комиссиях.

**Для MVP:** Можно начать с простого solver'а который:
- Сканирует интенты каждые 5 секунд
- Проверяет прибыльность через наш price oracle
- Исполняет через REF Finance
- Зарабатывает на разнице

**Для Production:** Нужна более продвинутая система с:
- Множественными solvers (конкуренция)
- Сложными маршрутами (multi-hop swaps)
- MEV защитой
- Оптимизацией gas costs
