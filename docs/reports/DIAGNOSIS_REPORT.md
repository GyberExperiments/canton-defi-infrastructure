## 🔍 КОРНЕВАЯ ПРИЧИНА

**Проблема:** Master-нода Kubernetes кластера (31.129.105.180) недоступна или k3s-server на ней не запущен.

**Последствия:**
1. K3s-agent на worker-ноде не может синхронизироваться с master-нодой
2. Kubernetes API недоступен, поэтому:
   - Service endpoints не обновляются
   - Traefik не может получить информацию о подах через Kubernetes API
   - Ingress правила не могут быть применены правильно
3. Поды работают, но недоступны через Service/Ingress, потому что:
   - Service не знает IP адреса подов (endpoints пустые)
   - Traefik не может направить трафик к подам

**Цепочка проблем:**
```
Master-нода недоступна
  ↓
Kubernetes API недоступен
  ↓
Service endpoints не обновляются
  ↓
Traefik не знает куда направлять трафик
  ↓
502 Bad Gateway для статических файлов и API
```
