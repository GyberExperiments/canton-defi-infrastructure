## 📊 Система памяти проекта

### Архитектура памяти

**MCP (Model Context Protocol)** - интегрированная система хранения знаний:

1. **Entities (Сущности)**:
   - `MEMORY_MANAGEMENT_RULES` - правила управления памятью v3.0
   - `CONTEXTUAL_MEMORY_SYSTEM` - система контекстуализации
   - `CURRENT_ISSUES_TRACKING` - отслеживание текущих проблем
   - `MEMORY_QUALITY_CONTROL` - контроль качества памяти

2. **4-фазный протокол инициализации**:
   ```
   Phase 1: Query Analysis (CoD-оптимизация)
   Phase 2: Strategic Search (PROJECT → USER → SHARED → SYSTEM)
   Phase 3: Semantic Execution (точный → семантический → граф)
   Phase 4: Post Learning (обновление метрик, связей)
   ```

3. **Иерархия контекстов**:
   - PROJECT_* (вес 1.0) - проектные знания
   - SHARED_PERSON_* (вес 0.8) - пользовательские паттерны
   - SHARED_KNOWLEDGE_* (вес 0.6) - domain best practices
   - SYSTEM_* (вес 0.4) - базовые знания

4. **Изоляция контекстов**:
   - Строгое разделение USER/PROJECT/SHARED
   - Access control на уровне entities
   - Логирование кросс-контекстных обращений

**Важно:** Система памяти хранится в MCP базе AI, **не в файлах проекта**.

3. **Следовать** 4-фазному протоколу инициализации
4. **Соблюдать** изоляцию контекстов
