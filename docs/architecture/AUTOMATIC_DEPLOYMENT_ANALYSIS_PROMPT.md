# 🔍 **ПРОМТ ДЛЯ АНАЛИЗА АВТОМАТИЧЕСКОГО РАЗВЕРТЫВАНИЯ**

## 📋 **ЗАДАЧА**

Проанализировать и исправить систему автоматического обновления GitHub Secrets и CI/CD пайплайна при изменении настроек в админ панели.

## 🎯 **ТРЕБОВАНИЯ**

### **Что должно работать:**
1. ✅ **Автоматическое обновление GitHub Secrets** при изменении настроек
2. ✅ **Автоматический запуск CI/CD пайплайна** после обновления секретов
3. ✅ **Автоматическое развертывание** изменений без ручного перезапуска
4. ✅ **Уведомления о статусе** развертывания

### **Текущие проблемы:**
- ❌ GitHub Secrets не обновляются автоматически
- ❌ CI/CD пайплайн не запускается
- ❌ Показывается уведомление о необходимости ручного перезапуска
- ❌ 500 Internal Server Error при сохранении настроек
- ❌ 429 Too Many Requests (rate limiting)

## 🔧 **ТЕХНИЧЕСКИЕ ДЕТАЛИ**

### **Файлы для анализа:**
- `src/app/api/admin/settings/route.ts` - основной API endpoint
- `src/lib/github-secrets.ts` - сервис GitHub Secrets
- `src/lib/security.ts` - безопасность и валидация
- `.github/workflows/deploy.yml` - CI/CD пайплайн
- `.github/workflows/deploy-minimal-stage.yml` - пайплайн для minimal-stage

### **Переменные окружения:**
- `GITHUB_TOKEN` - токен для GitHub API
- `GITHUB_REPO_OWNER` - владелец репозитория
- `GITHUB_REPO_NAME` - название репозитория
- `GITHUB_WORKFLOW_ID` - ID workflow для запуска
- `GITHUB_BRANCH` - ветка для развертывания

### **Ключевые функции:**
- `createGitHubSecretsService()` - создание сервиса
- `updateMultipleSecrets()` - обновление секретов
- `triggerWorkflow()` - запуск CI/CD
- `validateSettingsUpdate()` - валидация данных
- `checkRateLimit()` - rate limiting
