**Требование**: Найти все большие файлы и папки, безопасно очистить диск, НЕ повредив ценные данные.

## 📋 ИНСТРУКЦИИ ДЛЯ AI АССИСТЕНТА

### ЦЕЛЬ
Безопасно освободить место на диске Mac OS, найдя и удалив только ненужные файлы (кэши, временные файлы, старые логи, неиспользуемые образы Docker и т.д.), НЕ трогая важные данные проектов и пользовательские файлы.

### ПРИНЦИПЫ РАБОТЫ

1. **ВСЕГДА СПРАШИВАТЬ ПЕРЕД УДАЛЕНИЕМ** - если есть хоть малейшее сомнение, что файл может быть важен
2. **НЕ УДАЛЯТЬ БЕЗ ПОДТВЕРЖДЕНИЯ** - показывать список файлов и спрашивать разрешение
3. **ПРИОРИТЕТ БЕЗОПАСНОСТИ** - лучше не удалить что-то, чем удалить важное
4. **ДОКУМЕНТИРОВАТЬ** - записывать что было удалено и почему

## 🔍 ПЛАН ДЕЙСТВИЙ (выполнить пошагово)

### ШАГ 1: Анализ использования диска

```bash
# 1.1. Общая статистика диска
df -h

# 1.2. Топ-20 самых больших директорий в домашней папке
du -h -d 1 ~ | sort -hr | head -20

# 1.3. Топ-20 самых больших файлов в домашней папке
find ~ -type f -size +100M -exec ls -lh {} \; 2>/dev/null | awk '{print $5, $9}' | sort -hr | head -20

# 1.4. Размер Docker данных
du -sh ~/Library/Containers/com.docker.docker/Data 2>/dev/null
docker system df 2>/dev/null || echo "Docker не запущен"

# 1.5. Размер кэшей браузеров
du -sh ~/Library/Caches/* 2>/dev/null | sort -hr | head -10

# 1.6. Размер логов системы
sudo du -sh /var/log/* 2>/dev/null | sort -hr | head -10

# 1.7. Размер временных файлов
du -sh /tmp/* 2>/dev/null | sort -hr | head -10
du -sh ~/tmp/* 2>/dev/null | sort -hr | head -10 2>/dev/null || echo "Нет ~/tmp"
```

**Что проверить:**
- ✅ Какие директории занимают больше всего места
- ✅ Есть ли очень большие файлы (>1GB)
- ✅ Сколько места занимает Docker
- ✅ Размер кэшей браузеров и приложений
- ✅ Размер логов системы

### ШАГ 2: Поиск безопасных для удаления файлов

#### 2.1. Docker данные (если Docker не критичен сейчас)

```bash
# Проверить размер Docker
docker system df 2>/dev/null

# Показать что можно удалить
docker images --format "{{.Repository}}:{{.Tag}} - {{.Size}}" | head -20
docker ps -a --format "{{.Names}} - {{.Size}}" | head -20
```

**Что можно удалить:**
- ⚠️ Неиспользуемые образы Docker (старые версии)
- ⚠️ Остановленные контейнеры
- ⚠️ Build cache Docker
- ❌ НЕ удалять: образы, которые используются в production

#### 2.2. Кэши браузеров и приложений

```bash
# Размер кэшей
du -sh ~/Library/Caches/* 2>/dev/null | sort -hr | head -20

# Кэши браузеров (можно безопасно удалить)
du -sh ~/Library/Caches/com.google.Chrome* 2>/dev/null
du -sh ~/Library/Caches/com.apple.Safari* 2>/dev/null
du -sh ~/Library/Caches/com.microsoft.edgemac* 2>/dev/null

# Кэши Xcode (если не используется)
du -sh ~/Library/Developer/Xcode/DerivedData 2>/dev/null
du -sh ~/Library/Developer/Xcode/Archives 2>/dev/null

# Кэши npm/pnpm/yarn
du -sh ~/.npm 2>/dev/null
du -sh ~/.pnpm-store 2>/dev/null
du -sh ~/.yarn 2>/dev/null
```

**Что можно удалить:**
- ✅ Кэши браузеров (будут пересозданы)
- ✅ Кэши npm/pnpm/yarn (будут пересозданы при установке)
- ⚠️ Кэши Xcode (только если не используется)

#### 2.3. Временные файлы и логи

```bash
# Временные файлы пользователя
find ~/tmp -type f -mtime +7 2>/dev/null | head -20
find /tmp -user $(whoami) -type f -mtime +7 2>/dev/null | head -20

# Старые логи (старше 30 дней)
find ~/Library/Logs -type f -mtime +30 -ls 2>/dev/null | head -20

# Старые файлы загрузок (старше 90 дней)
find ~/Downloads -type f -mtime +90 -ls 2>/dev/null | head -20
```

**Что можно удалить:**
- ✅ Временные файлы старше 7 дней
- ✅ Логи старше 30 дней
- ⚠️ Старые файлы из Downloads (только после подтверждения)

#### 2.4. Старые бэкапы и снимки Time Machine

```bash
# Локальные снимки Time Machine
tmutil listlocalsnapshots / 2>/dev/null
tmutil listlocalsnapshotdates / 2>/dev/null

# Размер снимков
sudo du -sh /.MobileBackups 2>/dev/null
```

**Что можно удалить:**
- ⚠️ Старые локальные снимки Time Machine (только если есть внешний бэкап)

#### 2.5. Большие файлы в проектах

```bash
# Найти node_modules в проектах (могут быть большими)
find ~/GYBERNATY-ECOSYSTEM -name "node_modules" -type d -exec du -sh {} \; 2>/dev/null | sort -hr | head -10

# Найти .next папки (Next.js build artifacts)
find ~/GYBERNATY-ECOSYSTEM -name ".next" -type d -exec du -sh {} \; 2>/dev/null | sort -hr | head -10

# Найти большие файлы в проектах
find ~/GYBERNATY-ECOSYSTEM -type f -size +500M -ls 2>/dev/null | head -20
```

**Что можно удалить:**
- ⚠️ node_modules в проектах (можно переустановить через npm/pnpm install)
- ⚠️ .next папки (можно пересобрать через npm run build)
- ❌ НЕ удалять: исходный код, конфигурационные файлы, .git папки
