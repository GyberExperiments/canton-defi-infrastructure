# 🔧 **ОТЧЕТ О ИСПРАВЛЕНИИ ВВОДА ЦИФРЫ 0 В ПОЛЯХ ЦЕН**

## ❌ **ПРОБЛЕМА**

Пользователь сообщил о проблеме с вводом цифры 0 в поля цен:
- "я пытаюсь ввести 0.22 на цену продажи но 0 не вводится"
- "причем 1-2-3 и другие цифры вводятся а 0 нет"

**Причина:** Проблема с логикой обработки `||` (OR) оператора в JavaScript, где `0` является falsy значением.

---

## ✅ **РЕШЕНИЕ**

### **1. Найдена корневая причина**

**Проблема в коде:**
```typescript
// Проблемный код:
value={editing ? editValues.cantonCoinSellPrice || '' : settings.cantonCoinSellPrice}
onChange={(e) => setEditValues(prev => ({ ...prev, cantonCoinSellPrice: parseFloat(e.target.value) }))}
```

**Объяснение проблемы:**
- `parseFloat("0")` возвращает `0`
- В JavaScript `0` является falsy значением
- `editValues.cantonCoinSellPrice || ''` возвращает `''` вместо `0`
- Пользователь видит пустое поле вместо `0`

### **2. Исправлена логика обработки**

**Исправленный код:**
```typescript
// Исправленный код:
value={editing ? (editValues.cantonCoinSellPrice ?? '') : settings.cantonCoinSellPrice}
onChange={(e) => setEditValues(prev => ({ ...prev, cantonCoinSellPrice: e.target.value ? parseFloat(e.target.value) : undefined }))}
```

**Ключевые изменения:**
- ✅ `||` → `??` (nullish coalescing) - различает `0` и `undefined`
- ✅ Добавлена проверка `e.target.value ?` перед `parseFloat`
- ✅ Возвращается `undefined` для пустых значений

### **3. Исправлены все проблемные поля**

**Исправленные поля:**
- ✅ `cantonCoinBuyPrice` - цена покупки
- ✅ `cantonCoinSellPrice` - цена продажи  
- ✅ `minAmount` - минимальная сумма

**Поля, которые уже работали правильно:**

## 🔍 **ТЕХНИЧЕСКОЕ ОБЪЯСНЕНИЕ**

### **Проблема с `||` оператором:**
```javascript
// Проблемный код:
const value = 0;
const display = value || ''; // Результат: '' (пустая строка)

// Правильный код:
const display = value ?? ''; // Результат: 0
```

### **Проблема с `parseFloat`:**
```javascript
// Проблемный код:
parseFloat(''); // Результат: NaN
parseFloat('0'); // Результат: 0 (falsy)

// Правильный код:
const value = input ? parseFloat(input) : undefined;
```
