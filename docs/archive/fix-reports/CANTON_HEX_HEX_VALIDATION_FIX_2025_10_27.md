# 🔧 Исправление валидации Canton адресов: Поддержка HEX::HEX формата

**Дата**: 27 октября 2025  
**Ветка**: `main`  
**Проблема**: Canton адреса формата `HEX::HEX` не проходили валидацию  
**Статус**: ✅ РЕШЕНО И ПРОТЕСТИРОВАНО

---

## 📋 Описание проблемы

### Симптомы
Пользователи не могли использовать валидные Canton адреса следующего формата:
```
04286df6fb621ddf3949a799a53e2fdc::1220da95ae5ff1535f5ca2d0797c1cc1a57c76b72bb7a0a2bc5f880eb61ed34f7ba8
```

Ошибка валидации: **"Invalid Canton address format"**

### Корневая причина
Существующие regex паттерны требовали, чтобы часть **ДО** двойного двоеточия (`::`) начиналась с **буквы**:

```typescript
// ❌ СТАРЫЙ ПАТТЕРН - НЕ РАБОТАЛ
/^[a-zA-Z][a-zA-Z0-9_]*::[a-fA-F0-9]{32,}$/
//  ^^^^^^^^ - требовал букву в начале
```

Но реальные Canton адреса могут иметь формат `participant_id::party_hint`, где обе части - **чистые HEX строки** (начинаются с цифры).

---

## 🔍 Исследование

### Анализ формата Canton адресов

Canton Network поддерживает **4 формата** адресов:

1. **HEX::HEX формат** (participant_id::party_hint) - **САМЫЙ РАСПРОСТРАНЕННЫЙ**
   ```
   04286df6fb621ddf3949a799a53e2fdc::1220da95ae5ff1535f5ca2d0797c1cc1a57c76b72bb7a0a2bc5f880eb61ed34f7ba8
   ```
   - Обе части - hex строки (могут начинаться с цифры)
   - Минимум 32 символа в каждой части
   - Разделитель: двойное двоеточие `::`

2. **Namespace формат** (с буквенным префиксом)
   ```
   bron::1220da95ae5ff1535f5ca2d0797c1cc1a57c76b72bb7a0a2bc5f880eb61ed34f7ba8
   ```
   - Начинается с буквы
   - Разделитель: `::`

3. **Классический формат** (name:fingerprint)
   ```
   alice::abcdef1234567890abcdef1234567890
   ```
   - Имя (буквы, цифры, дефисы, подчеркивания)
   - Разделитель: `::`

4. **Hex-only формат** (чистый fingerprint)
   ```
   1220da95ae5ff1535f5ca2d0797c1cc1a57c76b72bb7a0a2bc5f880eb61ed34f7ba8
   ```
   - Только hex символы, без разделителя
   - От 32 до 80 символов

### Реальные примеры из production
```
✅ 04286df6fb621ddf3949a799a53e2fdc::1220da95ae5ff1535f5ca2d0797c1cc1a57c76b72bb7a0a2bc5f880eb61ed34f7ba8
✅ aaaabbbbccccddddeeee111122223333::ffffeeeeddddccccbbbbaaaa99998888
✅ 12345678901234567890123456789012::98765432109876543210987654321098
```

## ✅ Решение

### Добавлен новый regex паттерн

```typescript
// ✅ НОВЫЙ ПАТТЕРН - ПОДДЕРЖИВАЕТ HEX::HEX
const CANTON_HEX_HEX_REGEX = /^[a-fA-F0-9]{32,}::[a-fA-F0-9]{32,}$/;
//                              ^^^^^^^^^^^^^ - любой hex символ (включая цифры)
```

### Приоритет проверки форматов

Валидация проверяет форматы в следующем порядке:

1. **HEX::HEX** (приоритет 1) - самый распространенный
2. **Namespace** (bron::...) - с буквенным префиксом
3. **Classic** (name::fingerprint) - именованный формат
4. **Hex-only** - без разделителя

```typescript
const isValidCanton = 
  CANTON_HEX_HEX_REGEX.test(trimmed) ||      // 1. Проверяем HEX::HEX
  CANTON_NAMESPACE_REGEX.test(trimmed) ||    // 2. Проверяем namespace
  CANTON_CLASSIC_REGEX.test(trimmed) ||      // 3. Проверяем classic
  CANTON_HEX_REGEX.test(trimmed);            // 4. Проверяем hex-only
```
