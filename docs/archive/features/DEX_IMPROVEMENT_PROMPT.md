# Промпт для улучшения NEAR Intents DEX интеграции

## Контекст проекта

Ты работаешь над улучшением интеграции NEAR Intents DEX в проекте **1OTC Exchange**. Проект построен на:
- **Next.js 15** (App Router)
- **TypeScript**
- **@near-js/client** для работы с NEAR RPC
- **NEAR Intents Protocol** для мультичейн обмена токенов
- Kubernetes deployment

**Текущее состояние:** Базовая интеграция реализована и работает через смарт-контракты NEAR Intents. Все пользовательские интерфейсы переведены на английский. Готово к тестированию в testnet.

## Задачи

### 1. Аудит текущей реализации

**Прочитай и проанализируй:**
- Официальную документацию NEAR Intents: https://docs.near-intents.org/near-intents
- Текущую реализацию в проекте:
  - `src/app/api/near-intents/swap/route.ts` - создание swap intent
  - `src/app/api/near-intents/bridge/route.ts` - создание bridge intent
  - `src/app/api/near-intents/status/[intentId]/route.ts` - получение статуса
  - `src/lib/near-intents.ts` - клиентская библиотека
  - `src/lib/near-wallet-utils.ts` - утилиты для подписания транзакций
  - `src/components/dex/SwapInterface.tsx` - интерфейс swap
  - `src/components/dex/BridgeInterface.tsx` - интерфейс bridge
  - `src/components/dex/NearWalletButton.tsx` - подключение кошелька
  - `docs/NEAR_INTENTS_INTEGRATION.md` - документация интеграции

**Выяви:**
- Несоответствия с официальной документацией NEAR Intents
- Ошибки в формате параметров контрактов (`create_swap_intent`, `create_bridge_intent`, `get_intent_status`)
- Проблемы с обработкой ошибок
- Отсутствующие валидации
- Проблемы с типами данных (BigInt, yoctoNEAR конвертация)
- Некорректные значения по умолчанию для контрактов (`verifier.testnet` vs реальные адреса)

### 2. Исправление критических багов

**Обязательно исправить:**

1. **Адреса контрактов:**
   - Уточнить точные адреса verifier контрактов для testnet и mainnet из документации
   - Заменить `verifier.testnet` на реальный адрес, если он указан в документации

2. **Формат параметров intent:**
   - Проверить точные названия полей в `intentData` согласно документации
   - Убедиться что используется правильный формат для `app_fees` и `fee_recipient`
   - Проверить правильность конвертации сумм в yoctoNEAR (или другие единицы для разных токенов)

3. **Обработка разных токенов:**
   - Текущая реализация использует `1e24` для всех токенов (предполагается NEAR)
   - Добавить правильную обработку decimals для разных токенов (USDT/USDC = 6, ETH = 18, NEAR = 24)
   - Реализовать правильную конвертацию сумм на основе decimals токена

4. **Gas и deposit:**
   - Проверить оптимальные значения gas для транзакций
   - Убедиться что `attachedDeposit` правильный (сейчас `'0'`)

5. **Подписание транзакций:**
   - Улучшить обработку ошибок в `near-wallet-utils.ts`
   - Добавить retry логику для неудачных подписаний
   - Улучшить fallback на redirect

6. **Получение статуса intent:**
   - Проверить корректность метода `get_intent_status` и его параметров
   - Добавить обработку ошибок если intent не найден

### 3. Разработка улучшенной версии

   - Обновлять UI при изменении статуса (pending → filled)

3. **История транзакций:**

4. **Улучшенная валидация:**

5. **Улучшенная обработка ошибок:**

   - Graceful degradation при недоступности сервисов

6. **UI улучшения:**

7. **Поддержка большего количества токенов:**
