# 📋 Полный список изменений: Исправление валидации Canton адресов

## 🎯 Цель
Исправить валидацию Canton адресов для поддержки формата `bron::` (namespace с двойным двоеточием)

## ✅ Измененные файлы исходного кода

### 1. `src/lib/utils.ts`
**Функция**: `validateCantonAddress()`

**Изменения**:
- ✅ Добавлена поддержка namespace формата `/^[a-zA-Z][a-zA-Z0-9_]*::[a-fA-F0-9]{32,}$/`
- ✅ Увеличена max длина hex с 64 до 80 символов
- ✅ Увеличена общая max длина с 85 до 150 символов

**Строки**: 56-80

## 🎯 Примеры работающих адресов

```javascript
// ✅ ВСЕ ЭТИ АДРЕСА ТЕПЕРЬ РАБОТАЮТ:

// 1. Namespace формат
"bron::122064322dfc38d58c6b0d4e5bb77d7ff91e20bbd9321c3dc4"
"canton::1220abcdef1234567890abcdef1234567890"

// 2. Классический формат
"alice:1234567890abcdef1234567890abcdef12345678"
"bob:abcdef1234567890abcdef1234567890abcdef12"

// 3. Hex-only формат
"1220dcd32eb92dd21a7418cfd3fe9f5bcaa3d88ced980004361ac57ca63db9f6275f"
"abcdef1234567890abcdef1234567890"

// 4. С пробелами (автоматически обрезаются)
" bron::122064322dfc38d58c6b0d4e5bb77d7ff91e20bbd9321c3dc4 "

// 5. Uppercase (case-insensitive для hex)
"BRON::1220ABCDEF1234567890ABCDEF1234567890"
```
