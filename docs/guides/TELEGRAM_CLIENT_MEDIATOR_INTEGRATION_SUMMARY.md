## 📊 АРХИТЕКТУРА

### Принципы интеграции:

1. **Единая точка управления** - медиатор управляет всеми каналами
2. **Ленивая инициализация** - Client API загружается только при необходимости
3. **Graceful degradation** - автоматический fallback на Bot API
4. **Не нарушает существующую логику** - все работает как раньше + новый канал

## 🔧 ИСПОЛЬЗОВАНИЕ

### Сценарий 1: Тейкер принял заявку

**Что происходит:**
1. Тейкер нажимает "Принять заявку" в клиентской группе
2. Медиатор получает callback
3. Статус обновляется: `pending` → `client_accepted`
4. **Медиатор вызывает `telegramClientService.notifyTakerAboutAcceptedOrder()`**
5. Тейкер получает сообщение от администратора с:
   - Order ID
   - Тип сделки (BUY/SELL)
   - Суммы (Payment/Receiving)
   - Цена CC
   - Адреса (Canton, Receiving)
   - Контакты (Email, Telegram инициатора)
   - Ссылка на чат с оператором

### Сценарий 2: Оператор отвечает клиенту

**Что происходит:**
1. Оператор нажимает "Ответить клиенту" в Telegram
2. Медиатор получает сообщение оператора
3. **Медиатор проверяет: есть ли `context.customerTelegram`?**
4. **ДА → отправляет через Telegram Client API** ✅
5. Клиент получает персональное сообщение от администратора
