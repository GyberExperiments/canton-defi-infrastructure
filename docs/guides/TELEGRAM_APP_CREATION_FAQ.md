# ❓ FAQ: Создание приложения в Telegram

**Дата**: 2025-01-27  
**Тема**: Часто задаваемые вопросы при создании приложения на my.telegram.org

---

## ❓ Что указать в поле URL при создании приложения?

### Ответ:

**Поле URL является опциональным** для получения сессии через GramJS.

### Варианты:

1. **Оставить пустым** (если поле не обязательное)
   - Для получения сессии URL не требуется
   - MTProto API не использует URL для авторизации

2. **Указать любой валидный URL** (если поле обязательное)
   - Например: `https://example.com`
   - Или URL вашего сайта: `https://your-domain.com`
   - Или URL проекта: `https://github.com/your-org/canton-otc`

### ⚠️ Важно:

- URL **не используется** для авторизации через MTProto API
- URL **не влияет** на работу Telegram Client API
- URL нужен только для веб-приложений (если создаете веб-версию)
- Для получения сессии через GramJS URL можно игнорировать

## ❓ Какие поля обязательны?

### Обязательные поля:

- ✅ **App title** - название приложения (любое)
- ✅ **Short name** - короткое имя (латиница, без пробелов)
- ✅ **Platform** - выберите `Other`

### Опциональные поля:

- ⚠️ **URL** - можно оставить пустым или указать любой URL
- ⚠️ **Description** - описание (можно оставить пустым)

## ❓ Что делать если появляется ошибка?

### Проблема: "ERROR" при создании приложения

**Решения:**

1. **Отключите VPN/Proxy**
   - IP должен соответствовать стране номера телефона
   - Используйте IP той же страны, где зарегистрирован номер

2. **Используйте чистую сессию браузера**
   - Откройте инкогнито/приватное окно
   - Отключите расширения браузера (особенно ad-blockers)
   - Очистите кеш и cookies для `my.telegram.org`

3. **Проверьте поля формы**
   - **App title**: только буквы (a-z, A-Z) и цифры, без пробелов и спецсимволов
   - **Short name**: только латиница и цифры, без пробелов, подчеркиваний, спецсимволов
   - Примеры:
     - ✅ `CantonOTCAdmin`
     - ✅ `canton_otc_admin`
     - ❌ `Canton OTC Admin` (пробелы)
     - ❌ `Canton-OTC-Admin` (дефисы)

4. **Укажите подробное описание**
   - Поле Description должно содержать развернутое описание
   - Например: `Admin client for Canton OTC Exchange notifications via Telegram Client API`

## ❓ Нужен ли URL для получения сессии?

### Ответ: **НЕТ**

URL **не требуется** для:
- ✅ Получения `api_id` и `api_hash`
- ✅ Авторизации через MTProto API
- ✅ Получения сессии через GramJS
- ✅ Отправки сообщений через Telegram Client API

URL нужен только для:
- ⚠️ Веб-приложений (если создаете веб-версию Telegram клиента)
- ⚠️ OAuth авторизации (если используете Telegram Login)

## ❓ Пример заполнения формы

### Рекомендуемые значения:

```
App title:     Canton OTC Admin Client
Short name:     canton_otc_admin
Platform:       Other
URL:            (оставить пустым или https://example.com)
Description:   Admin client for Canton OTC Exchange notifications via Telegram Client API
```

### После создания:

Вы получите:
- **api_id**: число (например, `12345678`)
- **api_hash**: строка (например, `abcdef1234567890abcdef1234567890`)

Эти данные нужно добавить в `.env.local`:
```bash
TELEGRAM_API_ID=12345678
TELEGRAM_API_HASH=abcdef1234567890abcdef1234567890
```

## ✅ ИТОГ

**Для получения сессии:**
1. Создайте приложение на https://my.telegram.org/apps
2. Поле URL можно оставить пустым или указать любой URL
3. Скопируйте `api_id` и `api_hash`
4. Добавьте в `.env.local`
5. Запустите `node scripts/setup-telegram-session.js`

**URL не влияет на работу!**
