# 💬 Настройка группы клиентов в Telegram

**Дата**: 3 декабря 2025  
**Статус**: ✅ РЕАЛИЗОВАНО

---

## 📋 Описание

Система поддерживает отдельную Telegram группу для клиентов. Сообщения от операторов автоматически отправляются в эту группу.

---

## ✅ Реализованный функционал

### 1. Отслеживание кредов для настройки группы

**Как работает:**
1. Создайте группу в Telegram для клиентов
2. Пригласите бота в группу
3. Сделайте бота администратором
4. Напишите боту в группе сообщение с текстом: **"ruheggs"**

**Что происходит:**
- Бот автоматически определяет ID группы
- Сохраняет ID группы в память
- Отвечает подтверждением в группе
- Логирует ID для постоянного хранения

**Ответ бота:**
```
✅ Группа клиентов настроена!

📋 ID группы: -1001234567890
💬 Сообщения клиентам будут отправляться в эту группу.

💡 Для постоянного хранения добавьте в env:
TELEGRAM_CLIENT_GROUP_CHAT_ID=-1001234567890
```

### 2. Автоматическая отправка сообщений в группу клиентов

**Когда отправляется:**
- Когда оператор отвечает клиенту через формат: `ORDER_ID: сообщение`
- Сообщение отправляется одновременно в:
  - ✅ Intercom (основной канал)
  - ✅ Telegram группа клиентов (если настроена)

**Формат сообщения в группе:**
```
💬 Сообщение от оператора

📋 Order ID: ORDER_ID
📧 Клиент: email@example.com

👤 Оператор: Имя оператора

💬 Сообщение:
Текст сообщения оператора
```

### 3. Обработка сообщений в группах

**Что обрабатывается:**
- Сообщения от бота в группах
- Сообщения с кредом "ruheggs" для настройки
- Сообщения операторов в формате `ORDER_ID: сообщение`

**Логирование:**
Все действия логируются с информацией о:
- ID чата
- Тип чата (group/supergroup/private)
- Название группы
- Отправитель сообщения

## 📊 Как получить ID группы

### Способ 1: Через бота
1. Добавьте бота [@userinfobot](https://t.me/userinfobot) в группу
2. Бот покажет ID группы

### Способ 2: Через API
```bash
curl "https://api.telegram.org/bot<BOT_TOKEN>/getUpdates"
```

Найдите в ответе `"chat":{"id":-1001234567890}`

### Способ 3: Через логи
После отправки сообщения "ruheggs" бот ответит с ID группы

## 🔍 Проверка работы

### 1. Проверить что группа настроена

**В логах должно быть:**
```
✅ Client group chat ID saved: -1001234567890
```

### 2. Отправить тестовое сообщение

1. Оператор пишет в группу операторов: `TEST-ORDER-123: Привет клиент!`
2. Проверить что сообщение появилось в группе клиентов
3. Проверить логи:
   ```
   ✅ Message sent to client group: {
     orderId: "TEST-ORDER-123",
     clientGroupChatId: "-1001234567890"
   }
   ```

### 3. Проверить уведомление оператору

Оператор должен получить:
```
✅ Ответ отправлен клиенту

📋 Order: TEST-ORDER-123
💬 Сообщение: "Привет клиент!"

📤 Каналы:
   ✅ Intercom
   ✅ Telegram группа клиентов
```

## 🐛 Отладка

### Проблема: Группа не настраивается

**Проверить:**
1. Бот добавлен в группу?
2. Бот является администратором?
3. Сообщение содержит "ruheggs"?
4. Логи содержат "Detected credentials message"?

### Проблема: Сообщения не отправляются в группу

**Проверить:**
1. `clientGroupChatId` установлен? (логи "Client group chat ID loaded")
2. Бот имеет права на отправку сообщений в группу?
3. Группа не удалена/заблокирована?
4. Логи содержат "Message sent to client group"?

### Проблема: ID группы теряется после перезапуска

**Решение:**
Добавить `TELEGRAM_CLIENT_GROUP_CHAT_ID` в environment variables
