# 🔍 Intercom VPN/Proxy Test Guide

## 🎯 Цель тестирования
Определить, блокируется ли Intercom по IP-адресу или есть другие причины проблем.

## 📊 Результаты текущего тестирования

### ✅ Что работает:
- **Все домены Intercom доступны** напрямую с вашего IP
- **API отвечает** (HTTP 406 - это нормально)
- **CSP настроен правильно** - все домены разрешены
- **Наш сайт работает** - HTTP 200

### 🔍 Вывод:
**Проблема НЕ в региональной блокировке!** Все домены Intercom доступны с вашего IP.

## 🧪 План тестирования через VPN

### Шаг 1: Установка VPN
```bash
# Рекомендуемые VPN сервисы:
# - ProtonVPN (бесплатный)
# - NordVPN
# - ExpressVPN
# - Surfshark
```

### Шаг 2: Тестирование
1. **Подключитесь к VPN серверу в другой стране** (например, США, Германия)
2. **Откройте наш сайт**: https://stage.minimal.build.infra.1otc.cc/
3. **Откройте консоль браузера** (F12 → Console)
4. **Обновите страницу** и посмотрите на ошибки

### Шаг 3: Анализ результатов

#### Если Intercom работает с VPN:
- ✅ **Проблема в региональной блокировке**
- 🔧 **Решение**: Настроить прокси для Intercom или использовать fallback

#### Если Intercom НЕ работает с VPN:
- ❌ **Проблема в другом** (CORS, CSP, браузерные настройки)
- 🔧 **Решение**: Исправить технические проблемы

## 🛠️ Альтернативные методы тестирования

### Метод 1: Браузерный тест
```bash
# Откройте в браузере:
http://127.0.0.1:8080/test-browser-intercom.html
```

### Метод 2: Curl тест
```bash
# Тест прямого подключения:
curl -I https://api-iam.intercom.io/messenger/web/launcher_settings

# Тест через прокси (если есть):
curl -I https://api-iam.intercom.io/messenger/web/launcher_settings --proxy socks5://127.0.0.1:1080
```

### Метод 3: Node.js тест
```bash
# Запуск нашего теста:
node test-intercom-proxy.js
```

## 🔧 Возможные решения

### Если проблема в региональной блокировке:
1. **Настроить прокси для Intercom**
2. **Использовать CDN с прокси**
3. **Настроить fallback механизмы**

### Если проблема в технических настройках:
1. **Исправить CORS политики**
2. **Обновить CSP заголовки**
3. **Проверить браузерные расширения**

## 📝 Инструкции для пользователя

### Для тестирования:
1. **Установите VPN** (ProtonVPN бесплатный)
2. **Подключитесь к серверу в США/Европе**
3. **Откройте наш сайт** и проверьте консоль браузера
4. **Сообщите результат**: работает ли Intercom с VPN

### Ожидаемые результаты:
- **С VPN**: Intercom должен работать без ошибок
- **Без VPN**: Могут быть ошибки блокировки

## 🎯 Следующие шаги

После тестирования с VPN:
1. **Если работает с VPN** → Настроить прокси для Intercom
2. **Если не работает с VPN** → Исправить технические проблемы
3. **В любом случае** → Улучшить fallback механизмы

---

**Важно**: Текущие тесты показывают, что проблема НЕ в региональной блокировке, а в технических настройках браузера или CORS политиках.
